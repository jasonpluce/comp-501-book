Lesson 2: Intro to Hardware & Software
======================================

--------------

Learning Objectives
-------------------

By the end of this lesson, students will be able to define hardware and
software, identify the core components of a computer (CPU, memory,
storage, input, and output devices), classify examples of system,
application, and utility software, and explain how hardware and software
work together through the flow of information (input → processing →
output).

Students will also describe the operating system’s role as a manager of
hardware resources and a bridge between users and machines, use
real-world metaphors to articulate the relationship between hardware and
software, and apply these ideas by analyzing everyday devices.

--------------

Materials Needed
----------------

Students need access to a computer with a projector or large display for
demonstrations. If possible, have a disassembled laptop photo or diagram
(physical poster or slide) that labels CPU, RAM, storage, battery,
cooling, and ports.

Prepare a few everyday devices for show-and-tell or images (smartphone,
game controller, printer). Sticky notes or index cards for quick checks
and categorization activities are helpful. If learning remotely, digital
slides with labeled images and a shared document or poll will substitute
for physical materials.

--------------

Lesson Outline
==============

1. Warm-Up: What Do We Mean by “Hardware” and “Software”?
---------------------------------------------------------

Open with a quick think-pair-share: **“Name one example of hardware and
one example of software from your phone, laptop, or game console.”**

Invite 2–3 students to share. Frame the lesson:

- **Hardware** is the physical machinery we can touch.
- **Software** is the set of instructions that tell the hardware what to
  do.

Note that both are essential—neither is useful alone.

--------------

2. What Is Hardware? Definition, Purpose, and Core Components
-------------------------------------------------------------

Define hardware as the tangible components that perform computation,
store data, and interact with the outside world.

Key components:

- **CPU (Central Processing Unit)** — the “brain” that executes
  instructions.
- **Memory (RAM)** — fast, short-term storage that holds active data and
  programs.
- **Storage (SSD/HDD)** — long-term storage for files and applications,
  even when power is off.
- **Input devices** — keyboards, mice, touchscreens, sensors, cameras,
  microphones.
- **Output devices** — monitors, printers, speakers, haptic motors.

**Inside a laptop:** Show or describe a labeled diagram: CPU, RAM, and
storage connected on the motherboard. Ask students:

- “Where would inputs enter?”
- “Where do outputs leave?”

Emphasize cooling and power delivery as supporting hardware.

--------------

3. What Is Software? Definition, Role, and Types
------------------------------------------------

Define software as the programs and instructions that run on hardware.

Types:

- **System software:** operating systems (Windows, macOS, Linux, iOS,
  Android) and drivers.
- **Application software:** browsers, word processors, games, messaging
  apps.
- **Utility software:** antivirus, backup/sync tools, compression
  utilities, disk cleanup.

**Everyday phone example:** Identify the phone OS, apps, and utilities.
Highlight how:

- Apps request services from the OS.
- The OS coordinates the hardware.

--------------

4. How Hardware and Software Work Together: Input → Processing → Output
-----------------------------------------------------------------------

Introduce the information flow loop:

**Input → Processing → Output → (feedback)**

1. **Input:** data enters via keyboard, mouse, touchscreen, microphone,
   or sensors.
2. **Processing:** CPU executes instructions; RAM stores working data;
   storage provides programs/files.
3. **Output:** results go to screen, speakers, printer, or actuator.

**Walkthrough (pressing a key):**

- Keyboard hardware sends a signal.
- Driver translates it for the OS.
- OS notifies the active application.
- CPU processes what to display.
- RAM temporarily stores data.
- Graphics system updates the monitor with the character.

--------------

5. The Operating System’s Role
------------------------------

The OS acts as both **traffic cop** and **translator**:

- **Managing resources:** CPU scheduling, RAM allocation, file
  organization, device access.
- **Providing services:** windows, menus, file dialogs, networking,
  security.
- **Acting as a bridge:** apps ask the OS to perform hardware operations
  via drivers.

A short demonstration (task manager, file manager) helps illustrate
these ideas.

--------------

6. Real-World Metaphors
-----------------------

- **Body & Mind:** hardware = body; software = mind.
- **Kitchen & Recipe:** hardware = kitchen/tools; software = recipe.
- **Instruments & Sheet Music:** hardware = instrument; software =
  music.

Invite students to propose their own metaphors.

--------------

7. Guided Practice: Identify Hardware & Software in Everyday Devices
--------------------------------------------------------------------

In pairs/small groups, students pick a device (smartphone, laptop, game
console, smart speaker, printer):

- List **three hardware parts** (CPU, camera, touchscreen).
- List **three software pieces** (OS, app, utility).
- Classify software as system/application/utility.
- Trace an **Input → Processing → Output** path (“take a photo,” “print
  a document”).

Groups share one insight. Early finishers may sketch a block diagram.

--------------

8. Optional Extensions
----------------------

- **Brief history:** from vacuum tubes & punch cards to microprocessors
  & app stores.
- **Emerging trends:** AI chips, cloud-based software, thin clients.

Discuss how these trends shift responsibilities between local and remote
hardware.

--------------

9. Assessment / Exit Ticket
---------------------------

Quick check (verbal or written):

- Define hardware and software in your own words.
- Circle which are hardware vs software: **battery, keyboard, antivirus,
  driver, display, word processor**.
- Describe the OS role in one sentence.
- Trace the I → P → O steps for “pressing Play in a music app.”

--------------

Homework / Extension
====================

Choose **one**:

1. **Device deep-dive:** List 5 hardware components and 5 software
   components (classify software). Explain one action using Input →
   Processing → Output.

2. **Short history snapshot (150–200 words):** Describe one milestone in
   hardware/software and its impact on everyday computing.

3. **Trends reflection (150–200 words):** Explain what an AI chip or
   cloud-based app is and how it may affect future purchases.

--------------

Evaluation
==========

Assess students based on:

- Accuracy in identifying hardware vs. software
- Clarity in describing core components
- Correct software classification
- Completeness of Input → Processing → Output explanation

**Rubric:**

- **Excellent:** precise definitions, correct classifications, clear
  I→P→O examples, accurate OS explanation.
- **Satisfactory:** generally correct with minor errors.
- **Needs Improvement:** unclear definitions, misclassification,
  incomplete explanations.

**Optional teacher reflection:** Note effective metaphors, usefulness of
visuals, and which devices were easiest/hardest for students.
